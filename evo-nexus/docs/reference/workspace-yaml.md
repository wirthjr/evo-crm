# workspace.yaml Reference

The `config/workspace.yaml` file is the central configuration for your EvoNexus instance. It's generated by `make setup` and read by the dashboard, scheduler, and setup scripts.

## Location

```
config/workspace.yaml
```

## Format

```yaml
name: "My Workspace"
owner: "Your Name"
company: "Your Company"
timezone: "America/Sao_Paulo"
language: "pt-BR"
port: 8080
```

## Fields

### name

**Type:** string
**Required:** yes

Display name for the workspace. Shown in the dashboard header and reports.

### owner

**Type:** string
**Required:** yes

Your name. Used in CLAUDE.md generation and report attribution.

### company

**Type:** string
**Required:** yes

Company name. Used in report headers and financial context.

### timezone

**Type:** string (IANA timezone)
**Required:** yes
**Default:** `America/Sao_Paulo`

Controls schedule timing for all routines. The scheduler uses this to determine when to run morning briefings, end-of-day reports, etc. Must be a valid [IANA timezone identifier](https://en.wikipedia.org/wiki/List_of_tz_database_time_zones).

Common values:
- `America/New_York` ‚Äî US Eastern
- `America/Chicago` ‚Äî US Central
- `America/Los_Angeles` ‚Äî US Pacific
- `America/Sao_Paulo` ‚Äî Brazil (BRT)
- `Europe/London` ‚Äî UK
- `Europe/Berlin` ‚Äî Central Europe
- `Asia/Tokyo` ‚Äî Japan

### language

**Type:** string (language code)
**Required:** yes
**Default:** `pt-BR`

Language for agent responses and generated reports. Agents use this to determine output language.

Common values:
- `pt-BR` ‚Äî Brazilian Portuguese
- `en` ‚Äî English
- `es` ‚Äî Spanish

### port

**Type:** integer
**Required:** yes
**Default:** `8080`

Port for the web dashboard. The Flask backend and React frontend are served on this port.

```bash
# Dashboard will be available at:
http://localhost:{port}
```

## How It's Used

| Component | Reads | Uses |
|-----------|-------|------|
| `make setup` | Generates the file | Interactive wizard |
| Dashboard backend | `port` | Server binding |
| Dashboard frontend | `name`, `owner` | UI display |
| Scheduler | `timezone` | Routine scheduling |
| setup.py (CLAUDE.md generation) | All fields | Context generation |

## Editing

You can edit the file directly:

```bash
nano config/workspace.yaml
```

Or re-run the setup wizard to regenerate it:

```bash
make setup
```

Changes to `port` require restarting the dashboard (`make dashboard-app`). Changes to `timezone` require restarting the scheduler (`make scheduler`).

## Companion Files

The setup wizard also generates these alongside `workspace.yaml`:

| File | Purpose |
|------|---------|
| `config/routines.yaml` | Routine schedules and enabled/disabled state |
| `.env` | API keys for integrations |
| `CLAUDE.md` | Context file for Claude Code sessions |
